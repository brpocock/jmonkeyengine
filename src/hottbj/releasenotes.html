Change List

  • Changes since 0.2d
    1. This product rebranded from "Blender ==> jME Exporter" to "HottBJ Exporter".
    2. Fix bug where mat rejections cause export to abort, regardless of settings.
    3. Improved animation/posing diagnostics
    4. Animation optimization/culling algorithm reinstated
    5. Fix to incorrect shader test (confused diffuse and specular shaders)
    6. Added detection and graceful omission of bad frame numbers and empty animations
    7. Fixed imperfect tests for non-effective matrix transformations
    8. Added new function to once-and-for-all format floats exactly as I want them written.
    9. Inverse value of stored "shiness" value, to accommodate jME's crazy implementation where the value actually indicates "unshininess".
    10. Now allow user to specify either Phong or Cook Torr specular shader. jME does nothing special with these, but these to are an indication that the user basically wants a "typical" specular look.
    11. Fixed bug with ambient color calculation.
    12. Fixed mistake in ambient color calculation description in online docs.
    13. Use the Blender TexFace option as a hook to allow for a jME material-overlay trick. This allows for overlaying a base material with an overlay texture where the base bleeds through transparent areas.
    14. Support automatic jME instantiation of Spatial subclasses by use of a Blender String game property of name "jme.implClass".
    15. Support automatic loading of all Blender game properties into jME UserData maps. See sample file src/hottbj/AppAttrNode.java.
    16. Refactored transform translations so that jME side will always get +Z-forward or -Y-forward Objects, depending on whether Rotate X is set.
    17. Fixed nasty bug with non-deforming-bone-children. Not this, as well as normal object nesting under the Armature Object work very consistently and precisely.
    18. The Mountpoint strategy was borne out of the fruits of the previous two items. As I write this, Ashton is uploading his new video featuring Mountpoints.
  • Changes since 0.2c
    1. Enhanced existing Java sample class, and added new sample to exemplify jME-side usage of animation exports.
    2. Support arbitrary start and end frame settings (except can't be within the range of the animations frame set).
    3. Many fixes and enhancements to online docs, including addition of the page you are now reading.
    4. Added support for Empty object types.
    5. Added support for Empty Mesh object types, including support for our material inheritance feature.
    6. Several TODO items taken care of, which verified proper handling of items such as no-deform bones, envelope options, arbitrary mixing of Loc + Rot + LocRot frames and channel selections. The net effect to the software is primarily removal of TODO items and warnings in the code.
    7. Documented requirements and suggestions for content of starting frames. If these suggestions are followed, users can freely optimize following frames without dangers which would otherwise be incurred.
  • Changes since 0.1
    1. General support for Blender Armatures. It would take too much work to list the individual features here. With a few significant exceptions, we extend our support for arbitrary object to the realm of armatures. See the Skin & Bones page for details on exporter support for Armatures.
