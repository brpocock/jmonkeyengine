Change List

  • Changes since 0.1
    1. General support for Blender Armatures. It would take too much work to list the individual features here. With a few significant exceptions, we extend our support for arbitrary object to the realm of armatures. See the Skin & Bones page for details on exporter support for Armatures.
  • Changes since 0.2c
    1. Enhanced existing Java sample class, and added new sample to exemplify jME-side usage of animation exports.
    2. Support arbitrary start and end frame settings (except can't be within the range of the animations frame set).
    3. Many fixes and enhancements to online docs, including addition of the page you are now reading.
    4. Added support for Empty object types.
    5. Added support for Empty Mesh object types, including support for our material inheritance feature.
    6. Several TODO items taken care of, which verified proper handling of items such as no-deform bones, envelope options, arbitrary mixing of Loc + Rot + LocRot frames and channel selections. The net effect to the software is primarily removal of TODO items and warnings in the code.
    7. Documented requirements and suggestions for content of starting frames. If these suggestions are followed, users can freely optimize following frames without dangers which would otherwise be incurred.
